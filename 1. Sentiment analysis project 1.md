## [[#📌 Вступ]]

Financial Sentiment Analysis — це AI/NLP (Natural Language Processing) проєкт для аналізу фінансових текстів та автоматичного визначення емоційного тону повідомлень. Основна задача системи полягає у класифікації financial news та market-related text на:

- positive,
- negative,
- neutral.
  
У сучасному фінансовому середовищі банки, інвестиційні компанії та аналітичні платформи постійно працюють з великим обсягом текстових даних:

- фінансовими новинами,    
- market reports,
- earnings reports,
- stock market commentary,
- banking updates.

Ручний аналіз такої кількості інформації є складним та неефективним, тому компанії все частіше використовують AI та NLP-системи для автоматичного аналізу настроїв ринку та виявлення потенційних ризиків.

Особливістю financial NLP є те, що фінансова мова значно відрізняється від звичайного тексту. Багато financial expressions мають специфічне значення, яке general-purpose NLP models можуть інтерпретувати неправильно. Наприклад, фраза:  
“<font color="#ff0000">losses</font> narrowed during Q2”  
містить слово “<font color="#ff0000">losses</font>”, але фактично несе позитивний зміст для бізнесу.

Основною метою проєкту є:

- побудова системи financial sentiment classification,
- порівняння classical NLP approaches та transformer-based models,
- аналіз ефективності domain-specific financial transformers.

У межах проєкту було реалізовано та порівняно декілька моделей:

- Naive Bayes,
- Logistic Regression,
- SVM,
- DistilBERT,
- FinBERT.

## [[#⚙️ Попередня обробка даних]]

Після проведення exploratory data analysis наступним етапом стала попередня обробка даних (data preprocessing). Основною метою preprocessing було підготувати textual data до подальшого використання у machine learning та transformer-based NLP models.

У задачах NLP якість preprocessing має критичне значення, оскільки навіть незначні помилки або шум у тексті можуть негативно впливати на результати classification models.

### [[# 🧹 Очищення даних]]

Першим етапом preprocessing було очищення dataset від некоректних або неповних значень.

Було виконано:

- видалення missing values,
- видалення зайвих пробілів,
- базове очищення textual data.

```python
df = df.dropna()

df["Sentence"] = (df["Sentence"].astype(str).str.strip())

df["Sentence"] = df["Sentence"].apply( 
	lambda x: x[0].upper() + x[1:] if len(x) > 0 else x)
```

У transformer-based NLP models надмірний preprocessing часто є небажаним. На відміну від classical NLP approaches, сучасні transformer models здатні самостійно працювати з punctuation, capitalization та більшістю semantic patterns.

Тому preprocessing був навмисно мінімальним, щоб зберегти original financial context sentences.

### [[#🔢 Кодування міток]]

Machine learning models не можуть працювати безпосередньо з textual labels, тому sentiment classes були перетворені у numerical representation.

Було використано таке mapping: 

```python
label_map = {
"negative": 0,
"neutral": 1,
"positive": 2
}

df["label"] = df["Sentiment"].map(label_map)
```

Label encoding дозволяє models працювати з classification targets у numerical form та є стандартним preprocessing step для supervised machine learning tasks.

### [[#✂️ Розділення навчання/тесту]

Для оцінки моделей dataset був поділений на:
- training set,
- testing set.
Було використано stratified split для збереження однакового розподілу sentiment classes у train та test subsets.

```python 
from sklearn.model_selection import train_test_split

train_df, test_df = train_test_split( 
df, test_size=0.2,
stratify=df["label"],random_state=42)
```

Використання stratified split є особливо важливим у задачах sentiment analysis через class imbalance. Це дозволяє уникнути ситуації, коли одна з sentiment categories буде недостатньо представлена у test dataset.

### [[#🚀 Підсумок попередньої обробки]

На етапі preprocessing було виконано:

- видалення missing values,
- очищення зайвих пробілів,
- корекцію першої літери речень,
- label encoding,
- train/test split.

Ці кроки підготували dataset до подальшого навчання та порівняння NLP-моделей.

### [[#🔵 Naive Bayes 🇺🇦]]

Naive Bayes є однією з найпоширеніших classical NLP models для задач text classification.

Модель базується на ймовірностях появи слів у різних classes та часто використовується як baseline model у NLP tasks.

```python
from sklearn.pipeline import Pipeline
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.naive_bayes import MultinomialNB
from sklearn.metrics import classification_report

nb_pipeline = Pipeline([("tfidf", TfidfVectorizer()),("model", MultinomialNB())])

nb_pipeline.fit(train_df["Sentence"],train_df["label"])
nb_preds = nb_pipeline.predict(test_df["Sentence"])

print(classification_report(test_df["label"],nb_preds))
```

### [[#🟢 Logistic Regression 🇺🇦]]

Logistic Regression є однією з найпопулярніших baseline models для NLP classification tasks.

У поєднанні з TF-IDF (Term Frequency — Inverse Document Frequency) vectorization модель здатна ефективно працювати з high-dimensional text data.

```python
from sklearn.linear_model import LogisticRegression

lr_pipeline = Pipeline([("tfidf", TfidfVectorizer()),
("model", LogisticRegression(max_iter=1000))])

lr_pipeline.fit(train_df["Sentence"],train_df["label"])

lr_preds = lr_pipeline.predict(test_df["Sentence"])

print(classification_report(test_df["label"],lr_preds))
```

### [[#🟠 DistilBERT 🇺🇦]]

DistilBERT є lightweight transformer model, створеною як компактніша та швидша версія BERT.

Модель використовує transformer architecture та semantic contextual understanding для sentiment classification.

```python 
from transformers import pipeline
from sklearn.metrics import classification_report

distilbert_model = pipeline("sentiment-analysis",
	model="distilbert-base-uncased-finetuned-sst-2-english")

distilbert_preds = []

for text in test_df["Sentence"]:
	result = distilbert_model(text[:512])[0]["label"]
	
	if result == "POSITIVE":
		distilbert_preds.append(2)
	else:
		distilbert_preds.append(0)

print(classification_report(test_df["label"],distilbert_preds))
```

![[Pasted image 20260528130139.png | align=center]]


DistilBERT є general-purpose NLP model та не була спеціально навчена на financial text. Це дозволило оцінити, наскільки general NLP models здатні працювати з domain-specific financial language.

### [[#🔴 FinBERT 🇺🇦]]

FinBERT є transformer-based model, спеціально натренованою на financial text та financial sentiment analysis tasks.

Модель адаптована до:
- financial terminology,
- market language,
- banking vocabulary,
- earnings reports,
- stock market commentary.

```python 
from transformers import pipeline

finbert_model = pipeline("text-classification",model="ProsusAI/finbert")

finbert_preds = []

for text in test_df["Sentence"]:
	result = finbert_model(text[:512])[0]["label"]

	if result == "positive":
		finbert_preds.append(2)
	elif result == "neutral":
		finbert_preds.append(1)
	else:
		finbert_preds.append(0)
		
print(classification_report(test_df["label"],finbert_preds))
```

![[Pasted image 20260528130438.png | align = center]]

FinBERT була використана як domain-specific NLP model для financial sentiment classification. На відміну від general NLP models, FinBERT краще розуміє специфіку financial language та semantic meaning financial expressions.

### [[#🔲 Аналіз матриці плутанини]]

Для детальнішого аналізу predictions було використано confusion matrix.

Confusion matrix дозволяє оцінити:
- які classes модель класифікує правильно,
- між якими classes виникає найбільше помилок,
- наскільки модель справляється з class imbalance.

```python
from sklearn.metrics import confusion_matrix
from sklearn.metrics import ConfusionMatrixDisplay
import matplotlib.pyplot as plt
# Confusion matrix
cm = confusion_matrix(test_df["label"],finbert_preds)
```

### [[#🧹 Czyszczenie danych]]

Wykonano:

- usunięcie missing values,

```python
df = df.dropna()

df["Sentence"] = df["Sentence"].apply(
```

W transformer-based NLP models nadmierny preprocessing jest często niepożądany. W przeciwieństwie do classical NLP approaches nowoczesne transformer models potrafią samodzielnie pracować z punctuation, capitalization oraz większością semantic patterns.

Dlatego preprocessing został celowo ograniczony do minimum, aby zachować original financial context sentences.

### [[#🔢 Kodowanie etykiet]]]

Label encoding umożliwia models pracę z classification targets w numerical form i jest standardowym preprocessing step w supervised machine learning tasks.

### [[#🧹 Data Cleaning]]

The first stage of preprocessing involved cleaning the dataset from incorrect or incomplete values.

The following steps were performed:

- removing missing values,
- removing unnecessary spaces,
- basic cleaning of textual data.

```python
df = df.dropna()

df["Sentence"] = df["Sentence"].apply(

```

In transformer-based NLP models, excessive preprocessing is often undesirable. Unlike classical NLP approaches, modern transformer models are capable of handling punctuation, capitalization, and most semantic patterns on their own.

Therefore, preprocessing was intentionally kept minimal in order to preserve the original financial context of the sentences.

### [[#🔢 Label Encoding]]

Machine learning models cannot work directly with textual labels, so sentiment classes were transformed into numerical representation.

The following mapping was used:

```python
label_map = {

"positive": 2
}

df["label"] = df["Sentiment"].map(label_map)
```

Label encoding allows models to work with classification targets in numerical form and is a standard preprocessing step for supervised machine learning tasks.

To evaluate the models, the dataset was divided into:

A stratified split was used to preserve the same distribution of sentiment classes in both the train and test subsets.
